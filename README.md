# Music Catalog API Test Automation\n\nComprehensive test automation framework for the Music Catalog Search API supporting Nugs.net and Grateful Dead catalogs.\n\n## 🎯 Features\n\n- **Complete API Testing**: Search, Release Changes, Favorites\n- **Catalog Access Validation**: Multi-catalog filtering and access rules\n- **Performance Testing**: Response time and load testing\n- **Contract Testing**: Schema validation and API contracts\n- **CI/CD Integration**: GitHub Actions with automated reporting\n- **Flexible Configuration**: Environment-based configuration\n- **Rich Reporting**: HTML reports with detailed test results\n\n## 🚀 Quick Start\n\n### Prerequisites\n- Python 3.8+\n- Git\n- Access to Music Catalog API\n\n### Installation\n\n1. **Clone Repository**\n```bash\ngit clone https://github.com/BohdanHryts/music-catalog-api-tests.git\ncd music-catalog-api-tests\n```\n\n2. **Set up Environment**\n```bash\npython -m venv venv\nsource venv/bin/activate  # On Windows: venv\\Scripts\\activate\npip install -r requirements.txt\n```\n\n3. **Configure API Settings**\n```bash\ncp .env.example .env\n# Edit .env with your API details\n```\n\n4. **Run Tests**\n```bash\n# Run all tests\n./scripts/run_tests.sh\n\n# Run specific test suites  \n./scripts/run_tests.sh smoke\n./scripts/run_tests.sh integration\n./scripts/run_tests.sh performance\n```\n\n## 📁 Project Structure\n\n```\nmusic-catalog-api-tests/\n├── src/\n│   ├── api_client/          # API client implementations\n│   ├── models/              # Data models and schemas\n│   └── helpers/             # Utility functions\n├── tests/\n│   ├── integration/         # Integration tests\n│   ├── contract/           # Contract tests  \n│   ├── performance/        # Performance tests\n│   └── conftest.py         # Test fixtures\n├── config/                 # Configuration management\n├── fixtures/              # Test data factories\n├── utils/                 # Test utilities\n├── reports/              # Generated test reports\n└── scripts/              # Utility scripts\n```\n\n## 🧪 Test Categories\n\n### Smoke Tests\nQuick validation of core functionality\n```bash\npytest -m smoke\n```\n\n### Integration Tests  \nEnd-to-end API functionality testing\n```bash\npytest tests/integration/\n```\n\n### Contract Tests\nAPI schema and contract validation\n```bash\npytest tests/contract/\n```\n\n### Performance Tests\nResponse time and load testing\n```bash\npytest tests/performance/\n```\n\n## 🔧 Configuration\n\n### Environment Variables\n```env\nBASE_URL=https://api.musiccatalog.com\nAPI_VERSION=v1\nAPI_KEY=your_api_key\nTIMEOUT=30\nMAX_RETRIES=3\n```\n\n### Test Markers\n- `smoke`: Critical functionality tests\n- `regression`: Regression test suite\n- `catalog`: Catalog-specific tests\n- `search`: Search functionality tests\n- `release_changes`: Release changes tests\n- `performance`: Performance tests\n\n## 📊 Reporting\n\nTests generate comprehensive reports:\n\n- **HTML Reports**: Detailed test results with screenshots\n- **Coverage Reports**: Code coverage analysis\n- **Performance Reports**: Response time metrics\n- **CI Reports**: Automated pipeline results\n\nReports are generated in `./reports/` directory.\n\n## 🔄 CI/CD Integration\n\nAutomated testing pipeline with GitHub Actions:\n\n- **Pull Request Validation**: Run on every PR\n- **Daily Regression**: Scheduled comprehensive testing\n- **Multi-Python Version**: Test against Python 3.8-3.11\n- **Performance Monitoring**: Track API performance over time\n\n## 🎵 Test Scenarios Covered\n\n### Catalog Access Rules\n- ✅ Single catalog filtering (nugs, playDead)\n- ✅ Multi-catalog access validation  \n- ✅ Shared content accessibility\n- ✅ Cross-catalog content rules\n\n### Search Functionality\n- ✅ Basic search across all categories\n- ✅ Personalized ranking (positions 6+)\n- ✅ Result limits (50 per category)\n- ✅ Search categories: Song, Album, Artist, Venue, Date, Year\n\n### Release Management\n- ✅ Status changes (Active/Inactive filtering)\n- ✅ Track-level metadata updates\n- ✅ Cross-catalog release updates\n- ✅ Optional field validation\n\n### Edge Cases\n- ✅ Unicode and special characters\n- ✅ Large result sets\n- ✅ Concurrent requests\n- ✅ Error handling and validation\n\n## 🤝
